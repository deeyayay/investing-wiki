## Investment Thesis

| Date | Status | Notes |
|------|--------|-------|
| 2026-06-12 | Initial | Candidate for L06 Memory tier; IPO March 2024, added to Nasdaq-100 May 2026 |

Astera Labs solves a critical bottleneck in GPU-accelerated AI: as model context windows and batch sizes expand beyond HBM capacity limits, hyperscalers face the "GPU memory wall." CXL memory expansion — pooling disaggregated DRAM into a shared fabric accessible by GPU clusters — provides an elegant architectural solution that Astera pioneered with its Leo Smart Memory Controller.

The Leo's deployment in Microsoft Azure's M-series VMs (earliest hyperscale CXL production deployment) represents the first commercial validation that CXL memory expansion works at scale, de-risking the entire ecosystem adoption curve. This first-mover advantage is immense: once Azure, Google Cloud, and AWS deploy CXL memory across their fleets, Astera's Leo becomes a required component in every next-generation AI cluster, creating a platform lock-in moat.

Bull case: Q1 2026 +93% YoY revenue growth on just one hyperscaler (Azure) suggests a multi-year multi-year TAM expansion as CXL adoption spreads. Scorpio CXL Fabric Switch (64-lane, 768 GB/s) enables rack-level memory pooling, expanding addressable TAM to include storage and memory networking. Gross margins of 72% (despite 15% of revenue R&D) indicate strong product-level economics with significant op margin upside as scale grows.

Risks: CXL ecosystem adoption delays if hyperscalers prefer proprietary solutions; competing memory technologies (optical interconnect, photonics-based memory fabric); customer concentration (80%+ Microsoft today).

**Key moat:** First-mover advantage + hyperscale production proof point + ecosystem network effects (more deployed CXL → more support momentum).

**Key risks:** Customer concentration, CXL adoption delays, hyperscaler in-house R&D.
